Autonomous Architecture • Knowledge • Documentation Governance System

* Diátaxis

# ARTIFACT METADATA TEMPLATE

Every generated artifact must have a metadata file:

`public/_meta/{artifact}.meta.json`

Example:

```
{
  "artifactName": "architecture_graph",
  "version": "1.0.0",
  "hash": "sha256:xxxx",
  "confidenceScore": 100,
  "confidenceModel": "architecture-ai",
  "createdBy": "architecture-scheduler",
  "createdAt": "timestamp",
  "reviewRequired": false,
  "previousVersions": []
}
```

---

# ARTIFACT VERSIONING RULES

Artifacts are versioned using **semantic versioning**.

```
major.minor.patch
```

Rules:

major
architecture structure changes

minor
new nodes / new documentation

patch
regeneration without structural changes

---

# CONFIDENCE SCORE GOVERNANCE

Every artifact generated by an AI phase must include:

```
confidenceScore
```

If:

```
confidenceScore < 90
```

Then:

```
reviewRequired = true
artifact moved to quarantine
entry added to review_queue.json
```

---

# ARTIFACT HASH

Artifacts must be canonicalized and hashed.

Algorithm:

```
SHA256
```

Purpose:

* corruption detection
* rollback verification
* integrity audit

---

# ROLLBACK POLICY

If a generated artifact breaks architecture integrity or reduces system integrity score:

```
restore previous artifact version from _archive
```

Rollback event recorded in:

```
public/architecture_audit.json
```

# SCHEDULER DAILY EXECUTION PROCEDURE

1 Read pipeline state

```
docs/automation/pipeline_state.yaml
```

Extract:

```
currentPhase
schedulerMode
confidenceThreshold
```

---

2 Determine phase

If

```
currentPhase > 16
```

then

```
currentPhase = 1
cycle += 1
```

---

3 Execute phase

All phases follow identical lifecycle.

```
validate inputs
execute analysis
generate artifact
calculate confidenceScore
generate metadata
validate artifact
commit OR quarantine
```
